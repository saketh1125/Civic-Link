# Civic-Link DPI - Database Schema

## Overview

Civic-Link uses **PostgreSQL 16** with **PostGIS 3.4** extension for geospatial capabilities. All geographic data uses the `Geography` type (not `Geometry`) with SRID 4326 for accurate earth-surface calculations.

## Table Specifications

### 1. users

**Purpose:** Commuter profiles and authentication

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| email | VARCHAR(255) | UNIQUE, NOT NULL | Corporate email address |
| hashed_email | VARCHAR(255) | UNIQUE, NOT NULL | SHA-256 hash of email |
| phone | VARCHAR(20) | NOT NULL | Mobile number with country code |
| gender | VARCHAR(20) | NOT NULL | 'male', 'female', 'undisclosed' |
| role | VARCHAR(20) | DEFAULT 'commuter' | 'commuter', 'driver', 'admin' |
| verification_status | VARCHAR(20) | DEFAULT 'pending' | 'pending', 'verified', 'rejected' |
| corporate_id | VARCHAR(100) | NULL | Company identifier |
| civic_score | DECIMAL(5,2) | DEFAULT 100.00 | Driver behavior score |
| civic_score_updated_at | TIMESTAMP | NULL | Last score update |
| is_active | BOOLEAN | DEFAULT TRUE | Account status |
| created_at | TIMESTAMP | DEFAULT NOW() | Account creation |
| updated_at | TIMESTAMP | DEFAULT NOW() | Last update |

**Indexes:**
- `idx_users_email` on `email` (unique)
- `idx_users_gender` on `gender` (for safety queries)
- `idx_users_corporate_id` on `corporate_id` (for company filtering)

### 2. commutes

**Purpose:** Driver ride offers

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| driver_id | VARCHAR(36) | FK users.id, NOT NULL | Driver offering the ride |
| origin | Geography(POINT,4326) | NOT NULL | Starting location (GPS) |
| destination | Geography(POINT,4326) | NOT NULL | Ending location (GPS) |
| origin_address | VARCHAR(255) | NOT NULL | Human-readable origin |
| destination_address | VARCHAR(255) | NOT NULL | Human-readable destination |
| departure_date | DATE | NOT NULL | Travel date |
| departure_time | TIME | NOT NULL | Travel time |
| seats_offered | INTEGER | DEFAULT 3 | Available passenger seats |
| seats_available | INTEGER | DEFAULT 3 | Remaining seats |
| is_women_only | BOOLEAN | DEFAULT FALSE | Women-only ride flag |
| max_walking_distance | INTEGER | DEFAULT 500 | Passenger walk limit (meters) |
| route_polyline | TEXT | NULL | Encoded route for mapping |
| status | VARCHAR(20) | DEFAULT 'active' | 'active', 'filled', 'completed', 'cancelled' |
| expires_at | TIMESTAMP | NOT NULL | Offer expiration time |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation timestamp |
| updated_at | TIMESTAMP | DEFAULT NOW() | Last update |

**Indexes:**
- `idx_commutes_driver` on `driver_id`
- `idx_commutes_origin_gist` GIST index on `origin` (for geospatial queries)
- `idx_commutes_destination_gist` GIST index on `destination`
- `idx_commutes_departure` on `departure_date, departure_time`
- `idx_commutes_status` on `status`
- `idx_commutes_is_women_only` on `is_women_only`

**PostGIS Queries:**
```sql
-- Find commutes within 500m of a point
SELECT * FROM commutes 
WHERE ST_DWithin(
    origin, 
    ST_SetSRID(ST_MakePoint(78.4020, 17.4930), 4326)::geography,
    500
);
```

### 4. commute_matches

**Purpose:** Successful driver-passenger pairings

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| commute_id | VARCHAR(36) | FK commutes.id, NOT NULL | Matched commute |
| commute_offer_id | VARCHAR(36) | FK commute_offers.id, NOT NULL | Matched offer |
| passenger_id | VARCHAR(36) | FK users.id, NOT NULL | Passenger |
| driver_id | VARCHAR(36) | FK users.id, NOT NULL | Driver |
| match_time | TIMESTAMP | DEFAULT NOW() | When match occurred |
| origin_distance_m | INTEGER | NOT NULL | Distance between origins (m) |
| destination_distance_m | INTEGER | NOT NULL | Distance between destinations (m) |
| commute_was_women_only | BOOLEAN | NOT NULL | Safety snapshot: commute flag |
| offer_was_women_only | BOOLEAN | NOT NULL | Safety snapshot: offer flag |
| status | VARCHAR(20) | DEFAULT 'pending' | 'pending', 'confirmed', 'completed', 'cancelled' |
| created_at | TIMESTAMP | DEFAULT NOW() | Creation timestamp |
| updated_at | TIMESTAMP | DEFAULT NOW() | Last update |

**Indexes:**
- `idx_matches_commute` on `commute_id`
- `idx_matches_offer` on `commute_offer_id`
- `idx_matches_passenger` on `passenger_id`
- `idx_matches_driver` on `driver_id`
- `idx_matches_status` on `status`

**Safety Snapshots:**
These fields preserve the women-only state at match time, preventing tampering:
- `commute_was_women_only`: Was the commute flagged women-only?
- `offer_was_women_only`: Was the offer flagged women-only?

### 5. civic_scores

**Purpose:** Driver behavior scoring history

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| user_id | VARCHAR(36) | FK users.id, NOT NULL | Driver being scored |
| score | DECIMAL(5,2) | NOT NULL | Civic score value |
| swerve_events_24h | INTEGER | DEFAULT 0 | Swerve events in 24h |
| speeding_events_24h | INTEGER | DEFAULT 0 | Speeding events in 24h |
| trip_count_24h | INTEGER | DEFAULT 0 | Trips completed in 24h |
| calculated_at | TIMESTAMP | DEFAULT NOW() | Score calculation time |

**Indexes:**
- `idx_civic_scores_user` on `user_id`
- `idx_civic_scores_calculated` on `calculated_at`

**Scoring Formula:**
```
S_new = (S_old × 0.85) + (max(0, 100 - (n_swerves × 5) - P_speeding) × 0.15)
```

### 6. commute_audit_logs

**Purpose:** Encrypted audit trail for all match actions

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| match_id | VARCHAR(36) | FK commute_matches.id | Related match |
| action_type | VARCHAR(50) | NOT NULL | 'match_created', 'match_cancelled', etc. |
| actor_user_id | VARCHAR(36) | FK users.id | Who performed the action |
| encrypted_payload | BYTEA | NOT NULL | AES-256-GCM encrypted data |
| created_at | TIMESTAMP | DEFAULT NOW() | When action occurred |

**Indexes:**
- `idx_audit_match` on `match_id`
- `idx_audit_created` on `created_at`
- `idx_audit_action` on `action_type`

**Encryption:**
- Algorithm: AES-256-GCM
- Key: From environment variable `AUDIT_LOG_ENCRYPTION_KEY`
- Plaintext includes: user IDs, locations (anonymized after 24h), timestamps

### 7. swerve_events (Telemetry)

**Purpose:** Lane-cutting events detected from IMU data

| Column | Type | Constraints | Description |
|--------|------|-------------|-------------|
| id | VARCHAR(36) | PRIMARY KEY | UUID generated by application |
| user_id | VARCHAR(36) | FK users.id, NOT NULL | Driver who swerved |
| match_id | VARCHAR(36) | FK commute_matches.id | Associated trip |
| gyro_z | DECIMAL(8,4) | NOT NULL | Z-axis rotation rate (rad/s) |
| magnitude | DECIMAL(8,4) | NOT NULL | |gyro_z| value |
| detected_at | TIMESTAMP | NOT NULL | When event occurred |

**Indexes:**
- `idx_swerve_user` on `user_id`
- `idx_swerve_detected` on `detected_at`
- `idx_swerve_match` on `match_id`

**Detection Criteria:**
- `abs(gyro_z) > 1.5 rad/s`
- 60-second cooldown between events

## Geospatial Considerations

### SRID 4326 (WGS 84)

- **Standard:** World Geodetic System 1984
- **Used by:** GPS, Google Maps, most mapping systems
- **Unit:** Degrees latitude/longitude
- **Conversion:** PostGIS automatically converts to meters for calculations

### Why Geography Instead of Geometry?

| Type | Calculations | Use Case |
|------|--------------|----------|
| Geometry | Cartesian (degrees) | Map projections, flat surfaces |
| Geography | Spherical (meters) | Real-world distances on Earth |

**Civic-Link uses Geography** because:
1. `ST_DWithin(origin, point, 500)` returns true if within 500 **meters**
2. No manual degree-to-meter conversion needed
3. Accurate for the ~10km KPHB to HITEC corridor

### Example Queries

```sql
-- Find commutes starting near KPHB Phase 3 (500m radius)
SELECT c.*, u.email, u.gender
FROM commutes c
JOIN users u ON c.driver_id = u.id
WHERE ST_DWithin(
    c.origin,
    ST_SetSRID(ST_MakePoint(78.4020, 17.4930), 4326)::geography,
    500
)
AND c.status = 'active'
AND c.seats_available > 0;

-- Calculate distance between two points
SELECT ST_Distance(
    ST_SetSRID(ST_MakePoint(78.4020, 17.4930), 4326)::geography,
    ST_SetSRID(ST_MakePoint(78.3770, 17.4430), 4326)::geography
) / 1000 AS distance_km;
-- Result: ~8.2 km
```

```sql
-- Enable extensions
CREATE EXTENSION IF NOT EXISTS postgis;
CREATE EXTENSION IF NOT EXISTS pgcrypto;
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- Create schema
CREATE SCHEMA IF NOT EXISTS civic_link;
SET search_path TO civic_link, public;
```

- PostGIS optimizations
- Connection settings
- Logging configuration

## Performance Optimization

### Indexing Strategy

1. **B-tree indexes:** For equality/range queries (status, dates)
2. **GIST indexes:** For geospatial queries (ST_DWithin)
3. **Composite indexes:** For multi-column filters

### Query Optimization

1. **selectinload:** Eager loading for related entities
2. **Bounding box filter:** Pre-filter before ST_DWithin
3. **Connection pooling:** asyncpg with 20+ connections
